## Permissions(권한)
> 전역 권한 설정 관리
> 프로젝트 별 권한은 각 프로젝트 설정에서 관리
![[Pasted image 20260527103641.png]]
### 1. File Permissions (파일 접근 권한)
- 에이전트가 내 컴퓨터에서 어떤 폴더/파일을 읽고(`Read`) 쓸 수 있는지(`Write`) 경로를 제한하는 규칙을 설정.
- Allow (허용), Ask (매번 묻기), Deny (차단)

### 2. Network Permissions (네트워크 접근 권한)
- AI 에이전트가 외부 인터넷 환경에서 읽어올 수 있는 URL 주소를 화이트리스트/블랙리스트로 지정.
- Allow (허용), Ask (매번 묻기), Deny (차단)

### 3. Terminal & Tooling Permissions (터미널 및 도구 권한)

- **Terminal Commands:**

    - 에이전트가 내 컴퓨터 터미널에서 실행할 수 있는 명령어 목록을 제어합니다. (예: `npm install` 이나 `git commit` 은 허용하고, 시스템 삭제 명령 등은 차단)
    
- **Commands Outside Sandbox:**
    
    - 기본적으로 에이전트는 격리된 안전 구역(Sandbox) 내부에서 작동하는데, 샌드박스를 벗어나 로컬 OS 환경에 직접적인 영향을 주는 명령어를 허용할지 여부를 설정합니다.
    
- **MCP Tools (Model Context Protocol 도구):**
    
    - 외부 개발 도구 및 서드파티 API를 Antigravity 에이전트와 직접 연동할 때 사용하는 프로토콜 관련 권한 설정입니다. 에이전트에게 더 강력한 외부 스킬(Skills)을 부여할 때 열어줍니다.

## **Appearance(외관 및 디스플레이 설정)**
![[Pasted image 20260527132106.png]]

### 1. Chat Settings (채팅 설정)

- **Verbose agent chat (상세 에이전트 채팅 출력):**
    
    - AI 에이전트가 최종 답변을 내놓기 전, 백그라운드에서 **추론하고 계획을 세우는 중간 단계(Thinking Steps)들을 화면에 실시간으로 전부 다 보여줄지** 결정하는 토글 스위치.
- 
    - **왜 중요할까?** 앞서 말씀하신 대원칙 중 'LLM이 실수했을 때의 대책'을 세우기 위한 모니터링 도구입니다. 에이전트가 엉뚱한 방향으로 생각의 삽질을 하거나 루프에 빠지는 것을 실시간으로 감시하다가 중간에 차단(Stop)할 수 있게 해줍니다.

### 2. Appearance & Themes (테마 및 디자인 설정)

전체적인 프로그램의 시각적 테마를 세밀하게 커스텀하는 공간입니다.

- **Appearance (디스플레이 모드):** * 전체 UI의 기본 모드를 선택합니다. 현재는 Dark(다크 모드)로 지정되어 있으며, `Light`나 OS 설정을 그대로 따르는 `System`으로 변경할 수 있습니다.
    
- **Light Theme (라이트 테마 커스텀):** * 밝은 화면 모드일 때의 프리셋(`Default Light`)과 배경색, 글자색(Foreground), 포인트 컬러(Accent)의 헥사코드(#)를 직접 수정할 수 있습니다.

## Model(모델 관리)
![[Pasted image 20260527151013.png]]
### 1. Model Credits (모델 크레딧 설정)

- **Enable AI Credit Overages (AI 크레딧 초과 허용):**
    
    - **역할:** 기본적으로 제공되는 요금제 기반의 모델 쿼터(Quota)를 모두 소진했을 때, 추가 결제해 둔 'AI 크레딧(보유 잔액)'을 사용해서 에이전트 작업을 끊김 없이 이어나갈지 결정하는 토글.



## 2. Model Quota (모델별 잔여 쿼터 현황)

각 AI 모델별로 내가 현재 얼마나 더 사용할 수 있는지 직관적인 게이지 바 형태로 보여줍니다. 플랜에 따라 주기적으로 리셋됩니다.

## Customizations(커스텀 기능 설정)
![[Pasted image 20260527151749.png]]
## 1. Token Usage (커스텀 토큰 사용량 관리)

- **역할:** 내가 추가한 프롬프트 규칙, 커스텀 스킬, MCP 서버 정보들이 AI 에이전트의 콘텍스트 창(Context Window) 용량을 얼마나 차지하고 있는지 보여주는 게이지입니다.
    
- **특징:** 안내문에 적혀있듯 이 버젯(Budget)을 초과하면 용량이 큰 커스텀 설정들이 자동으로 잘려 나갑니다(`truncated automatically`). 현재는 등록된 것이 없어서 `100.0%` 모두 사용 가능한 상태입니다.
    
- **대원칙 연결:** 앞서 말씀하신 'LLM의 부담 최소화'를 위해 콘텍스트가 가득 차지 않도록 시스템이 사전에 용량을 통제하는 가드레일입니다.
    

## 2. Installed MCP Servers (설치된 MCP 서버 리스트)

- **역할:** 우리가 심도 있게 대화 나눴던 **MCP 서버들을 연동하는 실제 메뉴**입니다.
    
- **조작:** 현재는 `No MCP Servers` 상태로 등록된 서버가 없습니다. 만약 아까 이야기한 'PostgreSQL 연동 MCP 서버'나 'GitHub 연동 서버'를 붙이고 싶다면 오른쪽의 **`Add MCP +`** 버튼을 눌러 서버 경로(Stdio 방식의 명령어 실행 경로)나 네트워크 주소(IP/URL)를 등록하면 됩니다.
    

## 3. Build With Google Plugins (구글 플러그인 연동)

- **역할:** 구글의 자체 서비스 생태계(예: Google Drive, Docs, Gmail, Maps 등)와 에이전트를 다이렉트로 연동할 수 있게 해주는 관리 메뉴입니다.
    
- **용도:** 오른쪽의 `Customize` 버튼을 눌러 플러그인을 활성화하면, 에이전트가 코딩을 하다가 내 구글 드라이브에 있는 기획서 문서를 읽어오거나 캘린더 일정을 확인하는 등의 연동 작업이 가능해집니다.

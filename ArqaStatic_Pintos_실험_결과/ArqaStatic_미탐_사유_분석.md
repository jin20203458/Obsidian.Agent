## 2. 🔍 정탐(True Positive)으로 재분류된 규격 위반 상세 분석 (54건)
비록 일반 C/C++ 표준 규격을 충족하고 런타임에 정상 동작할지라도, **국방무기체계 코딩 표준 규격에서 금지하거나 명시적 표현을 규정한 사항을 어긴 것은 모두 엄연한 정탐**입니다.

### ① `ast-function-call-argument-consistency` (배열 decay 및 형변환 누락) ➡️ **정탐**
* **대상 라인**: `synch.c` L134 `printf("Testing semaphores...");`, L135 `sema_init(..., 0);` 등
* **규격 근거**: [변환 26] 음수 값을 unsigned type으로 변환 금지 및 함수 타입 정합성 검사.
* **정탐 이유**: C 표준상 문자열 리터럴 `char[]`이 `const char *`로 자동 포인터 변환(Decay)되는 것이 허용될지라도, 코딩 표준은 타입 간 묵시적 변환을 배제하고 형 정합성을 명시할 것을 강제합니다. 특히 signed `int` 0을 `unsigned` 매개변수에 명시적 형변환 없이 넘겨준 것은 규칙 위반을 규격 그대로 짚어낸 정탐입니다.

### ② `ast-if-style` (else 블록 누락) ➡️ **정탐**
* **대상 라인**: `synch.c` L116, L218, L319
* **규격 근거**: [스타일 11] if - else if 문은 else 문도 반드시 포함 시킨다 (방어적 프로그래밍).
* **정탐 이유**: 실무 논리상 `else`일 때 아무 일도 하지 않더라도, 코딩 규격은 방어적 흐름 제어를 위해 명시적으로 빈 `else { }` 블록이라도 작성할 것을 요구하므로 완벽한 정탐입니다.

### ③ `ast-extern-function-declaration` (extern 한정자 누락) ➡️ **정탐**
* **대상 라인**: `synch.c` L50, L68, L71, L72 등 17건
* **규격 근거**: [스타일 07] 외부 함수 사용 시 이를 명시하고 사용해야 한다.
* **정탐 이유**: 소스 코드 내에서 외부 라이브러리 심볼 호출 시 명시적으로 `extern` 선언을 명시하지 않은 채 사용하는 코딩 습관을 단속한 규격 부합 정탐입니다.

### ④ `cfg-null-dereference-guard` (NULL 가드 생략) ➡️ **정탐**
* **대상 라인**: `synch.c` L155, L156 `sema_down (&sema[0]);`
* **규격 근거**: [포인터 35] 포인터는 참조전에 NULL 여부를 반드시 확인하여야 한다.
* **정탐 이유**: 인자로 넘어온 배열의 주소가 물리적으로 절대 NULL이 될 수 없을지라도, 해당 함수 내부 스코프에서 포인터 변수를 역참조할 때 **"사전 NULL 검사 분기나 ASSERT가 코드상 명시적으로 존재하지 않았다"**는 규격 자체를 위반했으므로 명백한 정탐입니다.

## 3. ❌ 기술적 오탐(False Positive) 항목 분석 (1건)

### ① `ast-incorrect-permission-assignment` (취약 권한 대입 오인)
* **대상 라인**: `synch.c` [L137](file:///C:/Users/user/Desktop/test_src/threads/synch.c#L137)
  ```c
  thread_create ("sema-test", PRI_DEFAULT, sema_test_helper, &sema);
  ```
* **오탐 이유**: 
  * 핀토스 커널의 스레드를 생성하는 `thread_create` 함수의 2번째 파라미터는 우선순위(Priority) 값인 `PRI_DEFAULT` (정수 31)입니다.
  * 그러나 도구는 이 함수명을 리눅스/윈도우의 파일/공유 자원 권한(Permission ACL) 할당 함수로 **도메인을 완전히 오인**하여, 우선순위 정수 값을 "취약한 권한 마스크 지정"으로 판단하여 경고를 띄웠습니다.
  * 이는 무기체계 코딩 표준 및 코드 논리를 벗어난 **도구 분석 엔진 자체의 식별 오류(100% 오탐)**가 맞습니다.

## 4. ❌ 미탐(False Negative) 항목 사유 분석 (1건)

### ① [포인터 32] 지역 스택 주소의 외부 유출 (`cond_wait` L298)
* **사유**: `ReturnStackAddress` 체커는 함수 반환 시점의 탈출(`return &local_var;`)만 감지하도록 국한되어 있으며, `void *` 캐스팅과 연결 리스트 매크로 내부로 전파되는 포인터 에일리어싱 흐름을 추적하지 못해 미탐되었습니다.

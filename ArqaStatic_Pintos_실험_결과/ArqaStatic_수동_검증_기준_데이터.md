# 0. ArqaStatic 수동 검증 기준 데이터 (threads/synch.c)

본 문서는 정적 분석 도구(Clang-Tidy)를 구동하기 전, 개발자가 직접 소스코드인 `threads/synch.c`를 대상으로 **국방무기체계 C/C++ 공통 및 C 전용 코딩 규칙 51종을 수동으로 전수 검증하여 판정해 놓은 기준 데이터(Golden Reference)**입니다.

엄격한 무기체계 코딩 규격 표준을 한 치의 융통성도 없이 대입하여 수동으로 규명해 낸 위반 개소들을 명시함으로써, 도구 분석 결과(정탐, 미탐, 오탐)의 수학적 정합성이 100% 일치하는 대조 분모를 형성합니다.

## 1. 📊 수동 검증 요약 (Manual Review Summary)

* **총 검증 대상 규칙:** 51종 (C/C++ 공통 46종 + C 전용 5종)
* **준수 판정 (Compliant):** **43종**
* **위반 판정 (Non-Compliant):** **8종 (총 57건 위반 개소 검출)**

### 🔎 수동 위반 검출 상세 명세 (8종 57건)
1. **[스타일 07] 외부 함수 사용 시 extern 명시 누락 (20건 위반):**
   * **사유**: 외부 파일/라이브러리 함수(`list_init`, `intr_disable` 등)를 호출하는 소스코드 전역에서 명시적 `extern` 프로토타입 선언이 부재한 상태로 호출함.
2. **[스타일 08] 한 줄에 하나의 명령문 작성 (5건 위반):**
   * **사유**: L60, L108, L123, L128, L147에서 함수 반환형과 함수 명칭을 개행 없이 한 줄에 기술하거나 선언 형식을 개행 없이 연달아 서술함.
3. **[스타일 11] if - else if 문에 else 블록 필수 (3건 위반):**
   * **사유**: L116, L218, L319에서 예외 제어 관리를 위한 빈 `else` 블록이라도 선언되어야 하는 방어적 프로그래밍 규격을 어기고 단독 `if` 블록으로 방치함.
4. **[조건식 19] 조건식 상수(Always Constant) 사용 금지 (16건 위반):**
   * **사유**: `for (i = 0; i < 10)` 진입 조건이나 멀티스레드 대기 조건 `while (sema->value == 0)` 등 정적 기호 연산 하에서 상시 참/거짓으로 굳어지는 조건식을 수정 없이 방치함 (경로 분석 포함 총 16개 개소).
5. **[변환 26] 음수 값을 unsigned 형식으로 변환 금지 및 형 정합성 누락 (8건 위반):**
   * **사유**: `unsigned` 매개변수를 기대하는 `sema_init` 등에 명시적 형변환 없이 signed `int` 상수(0, 1)를 대입하거나 배열 Decay 암묵 형변환을 발생시킴.
6. **[포인터 35] 포인터 참조 전 NULL 검사 누락 (2건 위반):**
   * **사유**: `sema_test_helper` [L155-L156](file:///C:/Users/user/Desktop/test_src/threads/synch.c#L155-L156)에서 `void *sema_` 인자에 대해 NULL 가드나 ASSERT 검증을 완전히 생략한 채 `&sema[0]`, `&sema[1]`로 즉시 역참조함.
7. **[포인터 32] 지역 스택 주소의 외부 유출 (1건 위반 - 도구 미탐):**
   * **사유**: `cond_wait` [L298](file:///C:/Users/user/Desktop/test_src/threads/synch.c#L298)에서 로컬 변수인 `waiter` 구조체의 스택 생명주기 주소를 외부 대기 리스트 `&cond->waiters`에 삽입 등록함.
8. **[C-5] 구조체/배열 선언 시 {} 초기화 의무 준수 누락 (2건 위반 - 도구 미탐):**
   * **사유**: `sema` 배열 [L131](file:///C:/Users/user/Desktop/test_src/threads/synch.c#L131) 및 `waiter` 구조체 [L290](file:///C:/Users/user/Desktop/test_src/threads/synch.c#L290) 선언 즉시 기본값 `{0}` 초기화를 누락함.

# 실전형 AGENTS.md 작성을 위한 핵심 프레임워크 가이드

## 1. AGENTS.md 기본 설계 원칙

- **용량 및 길이 제한:** 최상위 루트 파일은 에이전트의 연산 집중력을 유지하기 위해 **반드시 150라인 미만 또는 32 KiB 용량 이하로 간결하게 기획**해야 합니다. 거대 언어 모델에게 파일 전체의 자동 생성을 전담시키는 것은 지양해야 합니다.
- **점진적 공시 (Progressive Disclosure):** 복잡하고 상세한 아키텍처 규칙은 루트 파일에 전부 명시하지 않고, **하위 기술 폴더(예: `docs/`)의 개별 마크다운 문서로 계층화하여 링크 형태로 연결**해야 합니다.
- **명령어 완전성 명시:** 에이전트는 로컬 환경의 종속 변수를 알지 못하므로, **실행문 구성 시 생략 없이 완전한 구문(예: `pnpm lint && pnpm format:check`)을 제공**하여 빌드 검증 명령어의 복사 편의성을 극대화해야 합니다.
- **호환성 및 거버넌스 연동:**
    - 다중 환경(Claude Code 등) 통합 시 데이터 중복 관리를 피하기 위해 **심볼릭 링크(`ln -s AGENTS.md CLAUDE.md`)** 기법을 사용하거나 **메타 링크(`@AGENTS.md`)**를 상위 노출하여 상호 연동합니다.
    - 공통 용어는 `GLOSSARY.md`에, 에이전트 상호작용 및 잡담 방지 규칙은 `TEAM_PROTOCOL.md`와 같은 고차원 거버넌스 문서로 분리하여 병행 활용합니다.

## 2. 루트 디렉토리용 초경량 AGENTS.md 구조

리포지토리 최상단에 위치하는 핵심 설정 파일은 다음 5가지 섹션으로 구성하는 것이 표준입니다.

- **Overview (개요):** 프로젝트의 핵심 아키텍처(예: 모노레포 여부), 주요 기술 스택, 패키지 매니저 정보를 기입합니다.
- **Commands (명령어):** 전체 시스템 단위의 정적 분석, 린팅, 테스트 스위트 실행, 타겟 패키지 빌드 명령어를 나열합니다.
- **Progressive Guidance Links (점진적 안내 링크):** 타입스크립트 코드 표준, 테스트 및 모킹 환경, 데이터베이스 스키마 등 상세 도메인 규칙을 담은 **하위 문서 경로(예: `docs/TYPESCRIPT.md`)를 링크**합니다.
- **General Agent Rules (일반 행동 규칙):**
    - 실행 전 의존성 경계 확인 및 계획 수립 지시.
    - 수정 전 임포트 호출자 및 설정 파일 스캔 지시.
    - 무분별한 리팩토링을 금지하고, **타겟 도메인에 국한된 최소한의 외과 수술적 변경(Surgical Changes)** 수행 지시.
    - 작업 완료 선언 전 검증 스위트(Verification) 의무 실행 지시.
- **Boundaries (3티어 제약 경계):** 보안 및 오작동 방지를 위해 권한을 정량적으로 통제합니다.
    - **ALWAYS (항상 수행):** 커밋 전 로컬 린터 체크, API 하위 호환성 유지 등.
    - **ASK FIRST (사전 승인 필요):** 신규 패키지 추가나 시스템 공통 설정 변경 전 사용자 확인 등.
    - **NEVER (절대 금지):** `.env` 파일이나 API 키와 같은 민감 정보 커밋 금지, 수동 lockfile 수정 금지 등.

## 3. 서브 디렉토리(특화 도메인)용 패키지 구조

마이크로서비스나 특정 패키지 내부 단위에서 전역 정책을 오버라이드할 때 사용하는 로컬 패키지 가이드 구조입니다.

- **Overview:** 해당 패키지에 특화된 프레임워크와 데이터베이스 커넥터 정보를 명시합니다.
- **Local Execution Commands:** 상세 로깅 옵션이 포함된 로컬 단위 테스트, 테스트 컨테이너를 이용한 통합 테스트, 특정 코드 제너레이션 명령어 등 모듈 특화 명령어를 제공합니다.
- **Boundaries:**
    - 사용자 입력값 파라미터 검증 헬퍼 의무 사용 (ALWAYS).
    - 트랜잭션 롤백 구조 내부에 DB 변형(Mutation) 격리 (ALWAYS).
    - 운영 브랜치 스키마 불일치 방지를 위한 DB 푸시 전 확인 (ASK FIRST).
    - 필터링되지 않은 예외나 내부 에러의 클라이언트 노출 금지 (NEVER).

## 4. 코파일럿 커스텀 에이전트용 프론트매터 정의 사양

특정 도구에 에이전트를 연동할 때 상단에 명시하여 특화 역량을 표명하는 메타데이터 블록 구성입니다.

- **메타데이터 선언:** 에이전트의 `name`, `description`, 구동할 `model`, 사용할 외부 `tools`(예: 터미널 명령어 실행 등)를 규정합니다.
- **Persona (페르소나):** 에이전트의 전문적 역할(예: 보안 취약점 분석 및 안전한 코딩 검증자)을 강제합니다.
- **Core Directives (핵심 지시 사항):** 파일별 스캔을 통한 인젝션 표면 검사, 스키마 제약 준수 여부 확인, PR 승인 전 보안 린터 강제 실행 등 특화된 목적을 달성하기 위한 구체적 행동 규범을 나열합니다.

구글 안티그래비티 2.0(Google Antigravity 2.0)의 다중 에이전트 오케스트레이션 및 동적 스킬(Dynamic Skills) 아키텍처에 맞추어, 기존 정적 마크다운(`AGENTS.md`) 방식의 가이드를 전면 재구성한 프레임워크 최적화 지침입니다.

## 1. 안티그래비티 2.0 기본 설계 원칙

- **동적 컨텍스트 적재 (Dynamic Context Loading):** 고정된 마크다운 파일을 파싱하는 대신, 작업 컨텍스트(예: 프론트엔드 빌드, DB 마이그레이션)에 따라 필요한 '스킬 모듈'만 에이전트 메모리에 동적으로 주입합니다. 이를 통해 수만 토큰의 컨텍스트 포화(Context Saturation) 및 환각 현상을 방지합니다.
    
- **다중 에이전트 분산 (Decentralized Multi-Agent):** 단일 에이전트에게 모든 규칙(Lint, Test, DB 등)을 강제하지 않습니다. 매니저(Orchestrator) 에이전트가 요청의 성격을 분석한 후, 권한과 스킬이 제한된 전문 서브 에이전트(예: `Security_Agent`, `DB_Agent`)에게 작업을 위임합니다.
    
- **명령어 완전성 및 도구 바인딩:** 에이전트가 로컬 환경을 추론하지 않도록 완전한 실행 구문을 제공하는 원칙은 유지하되, 텍스트 가이드가 아닌 안티그래비티 SDK의 **Tool Call** 속성으로 명시하여 에이전트가 직접 터미널이나 파이프라인에서 실행할 수 있도록 구성합니다.
    

## 2. 글로벌 오케스트레이터 설정 (루트 `AGENTS.md` 대체)

리포지토리 최상단에는 텍스트 룰북 대신, 에이전트 매니저가 전체 시스템을 파악하고 라우팅하기 위한 설정 파일(예: `antigravity.yaml` 또는 `orchestrator.py`)이 위치해야 합니다.

- **Environment Overview (환경 개요):** 모노레포 구조, 사용 중인 패키지 매니저(예: pnpm), 주요 기술 스택을 정의하여 매니저 에이전트가 프로젝트 전체의 의존성 경계를 인식하게 합니다.
    
- **Agent Routing (에이전트 라우팅 규칙):**
    
    - 특정 디렉토리나 확장자에 접근할 때 어떤 서브 에이전트를 호출할지 지정합니다. (예: `src/db/` 수정 시 `DB_Mutation_Agent` 호출)
        
- **Global Boundaries (전역 3티어 제약 경계):**
    
    - **ALWAYS (자동 수행):** 모든 코드 커밋 전 보안 린터(Security Linter) 실행 및 하위 호환성 체크 의무화.
        
    - **HITL (Human-In-The-Loop / 사전 승인):** 시스템 공통 설정 파일(`package.json`, `.env.example`) 변경 또는 외부 API 키를 요구하는 도구 사용 시 사용자(Manager View)의 명시적 승인 대기.
        
    - **NEVER (하드 블록):** 민감 정보가 포함된 파일 수정 및 시스템 lockfile의 수동 변형을 API 레벨에서 차단.
        

## 3. 도메인 특화 스킬 모듈 구조 (서브 디렉토리 규칙 대체)

각 마이크로서비스나 패키지 내부에서는 개별 마크다운 파일 대신, 해당 도메인에 접근하는 에이전트가 장착(Equip)할 스킬 모듈(Skill Modules)을 정의합니다.

- **Trigger Conditions (실행 조건):** 해당 스킬이 주입되어야 하는 상황(예: 단위 테스트 실패 시, 특정 코드 제너레이션 명령어 실행 시)을 정규식이나 이벤트 트리거로 정의합니다.
    
- **Local Action Policies (로컬 실행 정책):**
    
    - 해당 도메인에 국한된 최소한의 외과 수술적 변경(Surgical Changes)만 허용하도록 컨텍스트 범위를 제한합니다.
        
    - DB 트랜잭션 도메인의 경우, 롤백 구조 내부에 변형(Mutation)을 격리하는 검증 스크립트를 도구(Tool)로 제공하여 에이전트가 검증을 의무적으로 수행하게 합니다.
        
- **Error Handling:** 필터링되지 않은 예외가 발생할 경우, 에이전트가 자체 판단으로 임의의 코드를 작성하지 못하도록 차단하고 즉시 매니저 에이전트에게 컨텍스트를 반환하도록 설계합니다.
    

## 4. 서브 에이전트 페르소나 및 SDK 바인딩 (프론트매터 대체)

코파일럿 등에서 사용하던 마크다운 프론트매터는 Vertex AI 및 안티그래비티 파이썬 SDK를 통한 객체 인스턴스화로 대체됩니다.

- **Identity & Model Configuration:** 서브 에이전트의 이름, 역할(예: 보안 취약점 분석가), 그리고 작업 강도에 맞는 적절한 모델(예: 경량 작업은 Gemini Flash, 복잡한 추론은 Gemini Pro)을 개별 할당합니다.
    
- **System Directives (핵심 지시 사항):**
    
    - 에이전트가 주입받을 시스템 프롬프트입니다. 스키마 제약 준수 여부 확인, 인젝션 표면 검사 등 구체적인 행동 규범을 코어 프롬프트 레벨에 설정합니다.
        
- **Tool Registration (외부 도구 등록):** 해당 에이전트가 사용할 수 있는 로컬 쉘 명령어, 정적 분석기, DB 커넥터 등을 API 형태로 바인딩하여 안전하게 호출할 수 있는 인터페이스를 제공합니다.
